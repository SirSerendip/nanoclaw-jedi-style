# Value Network Game — User Feedback #1: Amy & Dawn (StarterStudio)

**Date:** April 16, 2026
**Testers:** Amy and Dawn (StarterStudio)
**Product:** Value Network Game (interactive ecosystem mapping tool)
**Verdict:** Mixed — core concept strong, execution needs refinement

---

## What Works

- Core idea of building interconnected ecosystem maps through AI coaching is strong
- Three-round progressive/iterative structure lets people develop thinking over time
- Visual interface concept is promising — especially toggling flows on/off to explore stakeholder relationships
- Interactive discovery could genuinely facilitate ecosystem teams

## Areas Needing Refinement

### Subject Matter Depth
- AI backend doesn't demonstrate deep expertise in technology/innovation ecosystem design
- Gaps in representing key stakeholder groups: no trade associations, limited corporate engagement, no media/communications circles
- Misunderstandings about core relationships and their function/role in the ecosystem
- **Recommendation:** Train the model on domain-leading frameworks: Brad Feld, Chris Heivly's *Build the Fort*, Victor Hwang's rainforest thinking, David Erlichmann's *Network of Networks*, Donella Meadows, and other foundational ecosystem literature
- Hypothesis needs to be ~95% there to help people *expand* thinking, not correct basic gaps
- For StarterStudio, it placed the organization at center when it should be about the founders they serve and how everyone contributes to that
- Domain expertise with different types of networks, hierarchies, and governance is important to establish upfront

### Conversational Constraints
- Feels like one chance to ask a question per round before tokens are exhausted
- Recommended moves lack subject matter expertise
- Limits the iterative refinement central to good ecosystem thinking
- **Need:** Back-and-forth dialogue with the coach — follow-up questions, drilling deeper, getting clarification — not a single shot

### Persistence Across Rounds
- Changes made in one round do NOT carry forward to the next
- Refinements get lost — users want to build on prior work, not reset

### Visual Clarity
- Bidirectional flows shown as two separate single-direction arrows adds cognitive load — should be clearer two-way representation
- First load issues: all-gray screen appeared, colorful lemonade blobs were off-screen in upper left corner (looked like it was still loading)

### User Agency
- Not obvious how users can directly add stakeholders or modify the map themselves
- Interface assumes all changes come through coaching questions — limits hands-on exploration

## Strategic Product Insight

> "This could be *phenomenal* for exploring what-if scenarios — 'What changes in our network if we strengthen corporate engagement?' or 'Where are our biggest gaps in relationships?' That's systems mapping + activation planning in one experience."

- But AI backend must understand ecosystems at the same level as users (or better) — not just use a generic framework
- Can be kept simple by looking only at the primary stakeholder level (not all individual orgs inside it)

## Recommended Reading / Frameworks to Integrate

- Brad Feld — startup ecosystem building
- Chris Heivly — *Build the Fort*
- Victor Hwang — Rainforest thinking
- David Erlichmann — *Network of Networks*
- Donella Meadows — systems thinking

# PDF Reader

## Quick start

```bash
pdf-reader extract report.pdf              # Extract all text
pdf-reader extract report.pdf --layout     # Preserve tables/columns
pdf-reader fetch https://example.com/doc.pdf  # Download and extract
pdf-reader info report.pdf                 # Show metadata + size
pdf-reader list                            # List all PDFs in directory tree
```

## Commands

### extract — Extract text from PDF

```bash
pdf-reader extract <file>                        # Full text to stdout
pdf-reader extract <file> --layout               # Preserve layout (tables, columns)
pdf-reader extract <file> --pages 1-5            # Pages 1 through 5
pdf-reader extract <file> --pages 3-3            # Single page (page 3)
pdf-reader extract <file> --layout --pages 2-10  # Layout + page range
```

Options:
- `--layout` — Maintains spatial positioning. Essential for tables, spreadsheets, multi-column docs.
- `--pages N-M` — Extract only pages N through M (1-based, inclusive).

### fetch — Download and extract PDF from URL

```bash
pdf-reader fetch <url>                    # Download, verify, extract with layout
pdf-reader fetch <url> report.pdf         # Also save a local copy
```

Downloads the PDF, verifies it has a valid `%PDF` header, then extracts text with layout preservation. Temporary files are cleaned up automatically.

### info — PDF metadata and file size

```bash
pdf-reader info <file>
```

Shows title, author, page count, page size, PDF version, and file size on disk.

### list — Find all PDFs in directory tree

```bash
pdf-reader list
```

Recursively lists all `.pdf` files with page count and file size.

# mermaid-render

Render any Mermaid diagram to a high-resolution PNG using `mmdc` — fully local, clean, no browser server tricks.

## Triggers

- "render this mermaid"
- "mermaid-ify [topic]"
- "visualize as mermaid"
- "turn this into a diagram"
- "show me a [Wardley/Futures Wheel/graph] of..."
- any code block with ```mermaid ... ```

## Core approach

Use `@mermaid-js/mermaid-cli` (`mmdc`) via `npx`, pointed at the native ARM Chromium at `/usr/bin/chromium`.
The bundled Puppeteer binary fails on this ARM container — the puppeteer config fixes it.

## Puppeteer config (write once, reuse always)

```bash
cat << 'EOF' > /tmp/puppeteer-config.json
{"executablePath": "/usr/bin/chromium", "args": ["--no-sandbox", "--disable-setuid-sandbox"]}
EOF
```

## Render workflow

### Step 1 — Write the .mmd file

```bash
cat << 'EOF' > /tmp/diagram-${SLUG}.mmd
MERMAID_CODE_HERE
EOF
```

### Step 2 — Render with mmdc

```bash
npx @mermaid-js/mermaid-cli \
  -i /tmp/diagram-${SLUG}.mmd \
  -o /tmp/diagram-${SLUG}.png \
  -p /tmp/puppeteer-config.json \
  -t dark \
  --scale 3
```

Key options:
- `--scale 3` → 3x resolution (HD). Use `--scale 4` for very large diagrams.
- `-t dark` → dark theme. Use `-t default` for white background.
- `-w 1400` → explicit width in pixels (useful for quadrantChart to avoid label clipping)
- `-b transparent` → transparent background

### Step 3a — Post to Slack via IPC (PREFERRED — after host patch applied)

```bash
echo "{
  \"type\": \"file_upload\",
  \"chatJid\": \"slack:C0AMR0DRDFG\",
  \"filePath\": \"/tmp/diagram-${SLUG}.png\",
  \"caption\": \"${CAPTION}\"
}" > /workspace/ipc/messages/upload-$(date +%s%N).json
```

### Step 3b — Fallback: post via catbox.moe (until IPC patch applied)

```bash
URL=$(curl -s -F "reqtype=fileupload" -F "fileToUpload=@/tmp/diagram-${SLUG}.png" https://catbox.moe/user/api.php)
mcp__nanoclaw__send_message "${CAPTION}\n\n${URL}"
```

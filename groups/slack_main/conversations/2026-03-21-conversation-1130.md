# /slack-upload — Send a local file to Slack

This skill uploads a local file to a temporary public host and delivers the URL via `send_message` so Slack renders it as a preview.

## Why this approach?

The Slack Bot Token is intentionally kept out of the agent container's environment for security. So direct Slack API calls from within the container are not possible. Instead, we upload the file to a public temporary host and share the URL — Slack auto-previews images from public URLs.

## Workflow

1. Verify the file exists
2. Upload the file to `0x0.st` (primary) or `transfer.sh` (fallback) via curl
3. Send the returned URL via `send_message` with a caption

## Step-by-step

### 1. Check file exists

```bash
ls -lh "<file_path>"
```

If the file does not exist, report an error and stop.

### 2. Upload to 0x0.st (primary)

```bash
curl -s -F "file=@<file_path>" https://0x0.st
```

This returns a plain-text URL like `https://0x0.st/AbCd.png`. Capture it.

If 0x0.st fails or returns an error (non-URL response), try the fallback:

### 3. Fallback: transfer.sh

```bash
FILENAME=$(basename "<file_path>")
curl -s --upload-file "<file_path>" "https://transfer.sh/${FILENAME}"
```

This returns a URL like `https://transfer.sh/abc123/filename.png`.

### 4. Validate the URL

Check the response starts with `https://` — if neither service worked, report failure.

### 5. Send to Slack

Use `mcp__nanoclaw__send_message` with:
- The URL on its own line (so Slack unfurls/previews it)
- A short caption describing what it is

Example:
```
📸 Screenshot of lemonde.fr

https://0x0.st/AbCd.png
```

## Arguments

The ARGUMENTS line contains the file path and an optional caption. Parse them:
- First token (or quoted string): the file path
- Remainder: the caption (default: "Here is the file")

If no file path is given, tell the user to provide one.

# /news-tracker — Latest News on Any Topic

Searches Google News RSS feeds across 50+ curated tech, science, and AI publications.
Returns up to 10 most recent articles from the past 7 days — max 1 per source.

## Trigger phrases

Use this skill when the user says things like:
- "latest news on X"
- "what's happening with X"
- "recent coverage of X"
- "news about X"
- "what's new in X"
- "any news on X this week"

## Workflow

### Step 1 — Extract topic and optional days

Parse the ARGUMENTS:
- **topic**: the subject to search (required)
- **days**: override the default 7-day window (optional, e.g. `--days 14`)

### Step 2 — Run the script

```bash
python3 /home/node/.claude/skills/news-tracker/tech-tracker.py "<topic>" --days 7
```

With custom window:
```bash
python3 /home/node/.claude/skills/news-tracker/tech-tracker.py "<topic>" --days <N>
```

### Step 3 — Present results

Output the script results directly — they are already formatted for chat.
Then add a one-line editorial note if a clear trend or surprise is visible in the headlines.

If no results found: suggest trying broader keywords or a longer `--days` window.
